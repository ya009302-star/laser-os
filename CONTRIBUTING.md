# CONTRIBUTING — 開発規則

laser-os は長期管理する研究ソフトウェアであり、開発の多くを AI 支援で行う。
本ファイルの規則は、人間・AI を問わずすべての変更に適用される。
**AI に開発を依頼する際は、このファイルを必ずコンテキストに含めること。**

## リポジトリポリシー (必須遵守)

1. **コア物理と GUI の分離**
   `cavsim.core` / `cavsim.analysis` / `cavsim.io` は numpy のみに依存し、
   GUI ライブラリ (PySide6, pyqtgraph 等) を import してはならない。
   逆方向 (GUI → core) の依存のみ許可。

2. **物理変更には必ずテスト**
   物理計算 (行列、素子モデル、ビーム伝搬、安定性判定など) に影響する
   すべての変更は、解析解または信頼できる文献値と照合する pytest を
   追加・更新しなければならない。テスト無しの物理変更はマージ禁止。

3. **物理定数・材料値の捏造禁止**
   未知の屈折率・分散係数・熱光学係数などを AI が推定値で埋めてはならない。
   不明な値は不明として扱い、ユーザーに出典の提供を求めること。

4. **材料データには source_status を付与**
   材料データベース (将来モジュール) の各エントリは
   `source_status: "literature" | "measured" | "estimated" | "unknown"`
   と出典情報を必須フィールドとして持つ。`estimated` / `unknown` の値を
   シミュレーションに使う場合は警告を出すこと。

5. **動作中の挙動の保全**
   既存テストがすべて合格する状態を維持する。意図的な挙動変更は
   CHANGELOG.md に記録し、対応するテストを更新すること。

6. **大規模書き換えの禁止**
   ユーザーが明示的に要求しない限り、ファイル全体の書き直し・
   アーキテクチャ変更・パッケージ名変更 (`cavsim` の改名を含む) を
   行ってはならない。変更は最小差分で行う。

## AI 支援開発の手順

1. 作業前に対象ファイルを実際に読むこと (記憶からの再生成禁止)
2. 変更は最小差分。無関係なリファクタリング・整形を混ぜない
3. 変更後に `python -m pytest tests/ -q` を実行し、結果を報告する
4. 物理式を追加・変更した場合は `docs/PHYSICS.md` に規約を追記する
5. 検証状況が変わった場合は `docs/VALIDATION.md` を更新する
6. バージョンに影響する変更は `CHANGELOG.md` に追記する
7. AI は実施していない検証を「検証済み」と記述してはならない

## コミット規約

- 1 コミット 1 目的。物理変更と GUI 変更は分ける
- メッセージは `module: 要約` 形式 (例: `cavsim.core: add thermal lens element`)

## テスト実行

```bash
pip install -e .          # または pip install numpy pytest (コアのみ)
python -m pytest tests/ -q
```

GUI はヘッドレス CI では起動できないため、GUI 変更時は import 可能性
(`python -c "import cavsim.gui.main_window"`) と実機での手動確認を行う。
