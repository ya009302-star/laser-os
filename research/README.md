# research/ — 研究データ層

laser-os は cavsim (共振器シミュレータ) だけでなく、研究に必要な
データ — 論文・材料・光学系・実験ログ・解析結果 — を出典をたどれる形で
統合する研究基盤を目指す。`research/` はその**データ層**である。

> **現在の状態: 枠組みのみ (段階的実装中)**
> 本ディレクトリ群は構想を構造として示すために設置した枠組みであり、
> 各サブディレクトリの中身 (実データ) は、対応する研究が進むにつれて
> 段階的に追加される。空のサブディレクトリは「未着手」を意味する。
> 何をいつ作るかは docs/ROADMAP.md に従い、着手は指示によって行う。

## データ設計の正典 (重要)

research/ 配下のすべてのデータ構造・規約は、以下を**正典**とする。
各サブディレクトリの README に書かれたスキーマは要約であり、
矛盾する場合は正典が優先する。

- `research/DATA_SCHEMA.md` — 全レコード型 (Material/OpticalElement/
  Paper/CavitySetup/ExperimentLog) と PhysicalValue 規約、データレベル、
  単位、バージョニングの完全定義。
- `docs/DECISIONS.md` — 設計判断記録 (ADR)。PhysicalValue 採用、
  3 データレベル、ABCD・符号・分散の各規約の決定理由。

この設計は姉妹プロジェクト laser-research-os で確定したものを移植した
(2026-06-13)。中核思想: **すべての物理値に出典・条件・信頼度を持たせ、
未知は null、AI が出した値は verified にしない。**

## コードとデータの分離 (設計原則)

laser-os は 3 層に分離する。混ぜてはならない (CONTRIBUTING 規則1・7)。

```
src/cavsim/   計算コード (物理エンジン・GUI)。numpy 等のみに依存
research/     研究データ (論文・材料・光学系・実験ログ・解析記録)
docs/         文書 (物理説明・検証状況・設計思想・調査)
```

研究データを `src/` の中に置いてはならない。コードが research/ の
データを読む場合も、コードは src/ 側に置き、データは research/ に置く。

## データパイプライン (長期構想)

```
論文 (litdb)
  → 抽出値 (source_status つき)
  → 材料DB (matdb) / 光学系DB (optdb)
  → シミュレーション (cavsim)
  → 実験ログ (expdb)
  → 予測-実測比較・次実験判断 (compare)
```

## サブディレクトリ

| ディレクトリ | 役割 | 状態 |
|---|---|---|
| litdb/ | 論文・datasheet・DOI・抽出元の管理 | 枠組みのみ |
| matdb/ | 材料データ (Yb:YAG, Yb:FAP 等)。source_status 必須 | 枠組みのみ |
| optdb/ | 光学素子 (ミラー/OC/CM/GTI/SESAM/レンズ/LD) と共振器セットアップ | 枠組みのみ |
| expdb/ | 実験ログ (発振条件・CW/QML・出力・スペクトル等) | 枠組みのみ |
| compare/ | 予測-実測比較と次実験判断の記録 | 枠組みのみ |

## 機密・実データの扱い (重要)

CONTRIBUTING 規則7 に従う。**実験生データ・論文 PDF・個人文書は
このリポジトリにコミットしない。** research/ にコミットしてよいのは:

- 各ディレクトリの README とスキーマ定義
- 出典情報・書誌情報 (DOI, タイトル, 著者) と source_status
- ダミー値のテンプレート
- 公開可能な解析記録・判断メモ

論文 PDF・測定生データ・装置固有の非公開値は、リポジトリ外で管理し、
research/ 内からは参照パス・書誌情報のみで指す。`.gitignore` が
`*.pdf` や `data/` 等を防護しているが、これに頼らず内容で判断すること。
